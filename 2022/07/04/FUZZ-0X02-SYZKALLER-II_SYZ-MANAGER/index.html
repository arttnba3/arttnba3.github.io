  3. 0x02.syz-manager
    1. RunManager()：启动前的初始化工作
      1. *VM 池：Pool
      2. *VM 接口：Interface
      3. 初始化 VM Pool
      4. 建立 crash 目录，启动 reporter
      5. 创建 Manager 实例并初始化，启动 PRC 服务器
      6. 启动 进度记录 协程
      7. 启动 执行数据记录 协程
      8. 启动 dashboard 协程 与 错误检测，进入下一阶段
    2. vmLoop()：整体工作流程
      1. 外层大循环：调配 vm 实例进行 fuzz 相关工作
        1. 中层循环：将 pending 队列中需要复现的 crash 加入到 reproducing 队列
        2. Crash 结构体：记录 crash 信息
        3. needRepro()：检查一个 crash 是否需要被复现
        4. 对剩余的 instance 进行处理
          1. 内层小循环I：可以复现且有剩余的 instance -> 进行复现
            1. crash 复现：Run() -> repro()
          2. 内层小循环II：不可以复现但有剩余的 instance -> 进行 fuzz
        5. 对应处理各 channel 数据
